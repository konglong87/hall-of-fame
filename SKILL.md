# 名人堂skills专家集合

> 聚合顶尖思维,自动路由专家

## 使用方式

```
/hall-of-fame "高考选专业"
→ 自动路由到张雪峰

/hall-of-fame "产品怎么增长"
→ 自动路由到张一鸣或MrBeast
```

## 核心流程

### Step 1: 路由判断

收到用户问题后:

**关键词匹配(第一层路由):**
1. 读取 `router_keywords.json` 加载路由规则
2. 扫描用户问题中的关键词
3. 统计各类别匹配次数
4. 计算置信度 = 匹配关键词数 / 该类别关键词总数
5. 如果置信度 >= threshold → 直接路由到该专家
6. 如果置信度 < threshold → 进入第二层LLM分类

**多专家场景处理:**
- 检测"产品+增长" → 检查是否有"视频"关键词
  - 有"视频" → MrBeast
  - 无"视频",有"App/算法" → 张一鸣
- 检测"创业" → 检查是否有"成本/技术"关键词
  - 有"成本/第一性原理" → 马斯克
  - 无,有"YC/融资" → Paul Graham

**LLM分类(第二层路由):**
- 使用router_keywords.json中的prompt模板
- 调用Claude进行分类判断
- 输出专家名称 + 理由 + 置信度

**Fallback策略:**
- 默认fallback专家: Paul Graham
- 适用场景: 通用人生建议、写作、思考方法等

### Step 2: 动态加载

确定专家名称 `{expert-name}-perspective` 后:

```
使用 Read tool 加载:
hall-of-fame/experts/{expert-name}-perspective/SKILL.md

检查文件是否存在:
- 存在 → 加载完整内容 → 进入Step 3
- 不存在 → fallback到Paul Graham → 加载 paul-graham-perspective/SKILL.md
```

### Step 3: 专家角色扮演

加载专家SKILL.md后:

1. **首次激活声明**(只说一次):
   ```
   ⚠️ 专家视角提醒

   您已激活 {专家名称} 视角。
   本专家框架基于公开信息提炼,非本人观点。
   建议适用范围有明确边界,详见专家SKILL.md的"诚实边界"章节。
   ```

2. **遵循专家框架回答**:
   - 使用专家的核心心智模型(5-6个模型)
   - 应用专家的决策启发式(8条规则)
   - 模拟专家的表达DNA(句式、词汇、节奏)
   - 执行Agentic Protocol(如需WebSearch获取最新数据)

3. **专家框架内容结构**:
   - 身份卡: 我是谁、起点、核心信念
   - 核心智模型: 5-6个模型,三重验证
   - 决策启发式: 8条规则
   - 表达DNA: 句式、词汇、节奏、幽默规则
   - Agentic Protocol: 问题分类 → 研究步骤 → 回答框架
   - 角色扮演规则: 如何激活、如何退出
   - 诚实边界: 适用范围、局限、调研来源

### Step 4: 长对话管理

- 同一对话session内,专家角色持续
- 用户说"切换专家"/"换一个角度"/"退出" → 回到Step 1重新路由
- 用户说"直接调用X专家" → 绕过路由,直接加载指定专家

## 注意事项

- 每位专家框架基于公开信息提炼,非本人观点
- 专家会使用WebSearch等工具获取最新数据(Agentic Protocol)
- 适用范围有明确边界,详见每位专家的"诚实边界"章节
- 动态加载避免context污染(单次调用约4500 tokens)

## 技术实现

- **Hook机制**: SessionStart自动注入hall-of-fame介绍
- **路由策略**: 关键词匹配(零成本) + LLM分类(低成本)
- **动态加载**: Read tool按需加载专家SKILL.md
- **性能优化**: 主Skill启动<500 tokens,调用时<4500 tokens
